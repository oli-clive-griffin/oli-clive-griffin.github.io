HTML vs. CSS

A website’s main purpose is to display information. But if the information isn’t displayed in a nice way, no-one is ever going to pay attention. Websites ideally are full of useful information, displayed in an aesthetically pleasing way that is intuitive, easy to read, and easy to navigate around.

Think of a website as a house. To build a house, we need a lot of raw materials, and we need to put them together. HTML represents all the raw materials. All together in one big pile. Technically, everything is there, but it’s pretty ugly, with all the parts just stacked on top of each other.

CSS is the blueprint. It details how to put the parts of the house together to build a beautiful house. Where the rooms will be, what colour the walls are, and where to place windows. 

The browser is represented by the builders. They get all the parts of the house and piece them together in such a way to create a beautiful house. Technically, the builders haven't added any parts but they have put them all together properly.



Control Flow and Loops

We use javascript to tell the browser what to do. However we can only tell it once. The computer can never come back and ask us, “what do I do now?” or “that didn’t work, what should I do instead?”. Because of this, we often need to give the computer a lot of instructions, using flow control and looping to deliver those instructions.

Imagine you’re at the gym and you want to do 3 sets of bicep curls. The ‘code’ or instructions for doing that, might look something like this: 

  1. Pick up the dumbbells
  2. Bending your arms at the elbows, raise the weights.
  3. If you can’t raise the weights put them back down and go to step 6
  4. Lower the weights back down, straightening your arms
  5. Repeat steps 2 through 4 until you put the weights down
  6. Rest for 2 minutes
  7. If you got less than 8 reps in your last set, lower the weight
  8. If you got more than 15 reps in your last set, increase the weight
  9. Repeat steps 1 through 7, 3 times

Since we don’t know how many reps we’re going to be able to do, we have to use conditional statements and looping to control the flow of the instructions. The conditional statement is step 3: “If you can’t raise the weights put them back down and go to step 6”. The looping is step 5: Repeat steps 2 through 4 until you put the weights down”. By combining a conditional statement with a loop, we create a very flexible set of instructions that change in response to the person doing the exercise. If we were to simply say, lift the weight 8 times, or even worse, if we wrote out lines 2 and 4, 8 whole times, we would not only have a far longer set of instructions than we need, we could also run into other problems. For example, what if the weight is too heavy, and after 6 reps, we simply can’t lift it anymore. We would be stuck and not know what to do, unable to complete the next instruction. Flow control eliminates this problem by repeating steps until certain condition is met, then moving on to the next step.



The DOM

If the HTML is the raw materials, CSS is the plan, and the browser is the builders, then the DOM is the finished house especially with regard to layout and structure. We can interact with the DOM using other programming languages like javascript. In the house you might say:

  1. go in the front door, 
  2. go to the kitchen, 
  3. open the top-left drawer, 
  4. grab a spoon

In a website, you might have some javascript that says:

  1. Go to the document
  2. Go to the body
  3. Go to the header
  4. Grab the title text

It’s important to notice that in each case, we are starting at the most general and, level by level, focussing in and getting more and more specific. This is because the DOM is arranged like a tree, with each node being within the last.



Accessing Data

When we traverse the DOM, usually, the data we access will be an array or an object. Two very common examples of this are the two methods .getElementById and .getElementsByClass. When we retrieve an element by it’s ID, we will be given the single element as an object. Because in HTML, only one element should ever have a single ID. data in objects are stored as key-value pairs, this means every piece of information stored in the object has a key (eg. couch) and a value (eg. color) to access the data in an object, we just need a key to ask for and we will be given the value associated with that key.

getElementById.couch.color = brown

However, when we get elements by class, we will be given an array of objects, as a class is intended to apply to multiple elements in html. These objects are not associated with a key, they are only accessible by number, 0 being the first object, 1 being the second, and so on. To access a specific object within an array, we input the corresponding number. To access a specific property of that object, we can then input a key (eg. color) and will be given the associated value.

getElementsByClass.loungeFurniture = [couch, coffeeTable, Shelf]
getElementsByClass.parent[0] = couch
getElementsByClass.parent[0].color = brown


Functions

When we want to perform the same operation over and over again, but with different inputs, it can be helpful to use a function. Think of a function as a pre-packaged set of code which you plug your own inputs into.

Function addTwo (num) {
Return num + 2
}

The code above creates a function called addTwo which, when given a number (num) adds 2 to it and then gives it back. This is very useful because we can now ‘call’ the function at any time we like, inputting whatever number we’d like.

addTwo(44)

The above code will return 46, as we’re feeding the number 44 into the function addTwo.

Functions are incredibly useful because they allow us to write code once, then execute it whenever we want without writing it all out again.
